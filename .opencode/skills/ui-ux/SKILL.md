# UI / UX Skill

## Goal

Every interface should feel

- clean
- modern
- elegant
- fast
- intuitive

The user should immediately understand how to use the application.

---

# Design Principles

Always prioritize

- clarity
- consistency
- simplicity
- accessibility
- usability

---

# Visual Hierarchy

Use

- clear headings
- adequate spacing
- card layouts
- visual grouping
- consistent typography

Never create crowded interfaces.

---

# Spacing

Use generous spacing.

Prefer

- whitespace
- padding
- margins

Avoid cramped layouts.

---

# Colors

Prefer

- neutral backgrounds
- one primary color
- one accent color

Use colors only to communicate

- success
- warning
- error
- information

Never rely only on color.

---

# Components

Prefer

- Cards
- Dialogs
- Drawers
- Tabs
- Tables
- Badges
- Tooltips
- Toasts

---

# Buttons

Every button should have

- clear action
- loading state
- disabled state
- hover state
- focus state

---

# Forms

Every form should

- explain what is required
- validate instantly when appropriate
- show friendly errors
- preserve user input

---

# Tables

Include whenever appropriate

- pagination
- search
- sorting
- filters

Empty tables must show an empty state.

---

# Loading

Never leave blank screens.

Use

- Skeletons
- Spinners only for short waits
- Progressive loading

---

# Empty States

Every empty page should explain

- why it is empty
- what the user can do next

---

# Error States

Errors should

- explain what happened
- suggest a solution
- allow retry

Never expose stack traces.

---

# Responsive Design

Must work well on

- Desktop
- Tablet
- Mobile

Mobile first whenever possible.

---

# Accessibility

Always

- WCAG friendly
- keyboard navigation
- sufficient contrast
- visible focus
- screen reader compatibility

---

# Animations

Prefer subtle animations.

Use

- fade
- slide
- scale

Avoid excessive animations.

---

# Professional Look

The final result should resemble products such as

- Linear
- Notion
- Vercel
- GitHub
- Stripe Dashboard

Aim for a polished SaaS-quality interface rather than a basic CRUD application.
